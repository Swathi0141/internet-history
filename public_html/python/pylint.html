PYLINT

Pylint executing on calculator

  • Installing Pylint
    • Go to where you can call python from (I.E command prompt or terminal)
    • You can check this by typing python3
    • But wherever you installed python, pip should be installed
    • To call pip, you run python3 -m pip
    • To get Pylint, you run python3 -m pip install pylint
    • If somehow you do not have pip, follow this link: PIP Install
    • Then simply follow the above steps!
  • Running Pylint
    • In your shell (command prompt or terminal), you go to the directory where your code is
    • Run pylint file path or file name
    • If you have four python scripts in a folder called hero, you would run pylint hero/
    • Or if you wanted to check one of those scripts called class.py, you would run pylint hero/class.py
    • And that's it!
    • Pylint will display what is wrong and where, so you will just have to follow the results
    • A useful line to add to your code for pylint is this: # pylint: disable=too-few-public-methods
    • That line allows you to have one function in a file and not have to deal with pylint throwing an error
    • If you get weird import errors, add this line: # pylint: disable=import-error
    • Otherwise, if you know the pylint errors are wrong, just use the Internet to find the pylint error supression lines

  • OOP Terms and Definitons
    • Factory - an object for creating other objects: a function or method that returns objects of a varying prototype or class from some method call
    • Instantiation - creation of an object
    • Object - an instantiation of a class. You can say an object is a variable with the datatype being a class
    • Class - blueprint for an object: it defines the methods and attributes of an object
    • Namespace - collection of currently defined symbolic names along with information about the object that each name references
      There are three types:
      • Built-In: these are in the default functions and variables that exists in any python setting such as print() and len()
      • Global: these are functions and variables that solely exist in a single python file, accessible throughout the whole script. These types of variables would be declared at the top of the file, directly after any import statements. The functions associated with this would be static and class
      • Local: these are primarily variables created in functions and are only accessible in those functions. Otherwise some functions could be in the local space if they are non-static and non-class
    • Constructor - prepares a new object for use, often accepting arguments or parameters used to set attributes or member variables
    • Fixture - only in tests; ensures that there is a well known and fixed environment in which tests are run so that results are repeatable
    • Unit Test - testing method by which individual units of source code are tested to determine whether they are fit for use
    • Static Method - functions that do not rely on an object they are attached to
    • Class Method - functions that can be called with a class or with an object. They are bound to a class rather than an object and takes itself, cls as the first parameter. Lastly, these functions can access or modify the class state.
    • Facade - masks actual functions into one line functions to hide what the code actual does from the user
    • Design Pattern Used - general repeatable solution to a commonly occurring problem in software design
    • Method - functions associated with the objects of the class they belong to
    • Property - provides an interface to instance attributes. It provides a getter that retrieves a value in a class and a setter that sets a value in a class
